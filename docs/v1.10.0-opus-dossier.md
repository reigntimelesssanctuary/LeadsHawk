# v1.10.0 — Opus dossier verification + strategic intelligence

**Status:** Spec'd 2026-05-27. Shipping in this session.

Builds on v1.9.2's feedback infrastructure and v1.9.4's verified signal-research path.

## Why this exists

Brand and product research are LeadsHawk's foundational asset — everything downstream (deep scan, live monitor, sales briefs) is bottlenecked by dossier quality. v1.9.x's Perplexity-only dossier is good but has three weaknesses Claude Opus is well-suited to fix:

1. **Generic marketing language slips in** ("market-leading", "trusted partner", "innovative")
2. **Flat confidence** — thin-evidence claims read the same as well-sourced ones
3. **No "what we don't know" awareness** — Perplexity writes confidently about gaps

Plus the dossier is missing structured strategic content the user would otherwise produce manually (ICP segmentation, buying-cycle scenarios, competitive plays).

## Architecture

```
researchBrand(brandId, { feedback? })
researchProduct(productId, { feedback? })

  ├── Stage 1 — Perplexity sonar-deep-research  [unchanged from v1.9.x]
  │     • full dossier + citations
  │     • writes to canonical columns (category, positioning, ...)
  │     • writes raw text to raw_dossier (audit)
  │
  ├── Stage 2 — Claude Opus verify + sharpen  [NEW, gated by setting]
  │     INPUT: Stage 1 dossier + citations + knowledge blob + feedback
  │     • strip generic marketing language
  │     • annotate confidence per field (high|medium|low)
  │     • produce "What we don't know" markdown
  │     OUTPUT: sharpened field values (overwrite canonical columns)
  │             + confidence_levels JSON + unknowns markdown
  │             + verified_dossier JSON (full audit)
  │
  └── Stage 3 — Claude Opus strategic intel  [NEW, gated by setting]
        INPUT: Stage 2's verified dossier + feedback
        OUTPUT: strategic_intel JSON with:
          • icp_segments[]
          • buying_cycle_scenarios markdown
          • competitive_plays markdown
```

If `brandResearchAdvanced` (or `productResearchAdvanced`) is `false`, Stages 2+3 are skipped — v1.9.x behaviour preserved. Anthropic API key not configured also short-circuits to Stage-1-only with a logged warning.

Stage 2 failure preserves Stage 1's output and skips Stage 3. Stage 3 failure preserves Stage 2's output. Partial success is always better than rollback.

## Schema additions (idempotent)

On both `brands` and `products`:

- `raw_dossier TEXT NULL` — Stage 1's raw output for audit when Stage 2 overwrote canonical fields
- `verified_dossier TEXT NULL` — Stage 2 output JSON (full record of what Stage 2 wrote)
- `confidence_levels TEXT NULL` — JSON `{ field_name → 'high' | 'medium' | 'low' }`
- `unknowns TEXT NULL` — markdown bullets
- `strategic_intel TEXT NULL` — Stage 3 output JSON
- `last_advanced_research_at TEXT NULL` — timestamp; null until Stage 2+3 successfully run

## Settings

- `brandResearchAdvanced: boolean` (default `true`) — enables Stages 2+3 on brand research
- `productResearchAdvanced: boolean` (default `true`) — enables Stages 2+3 on product research

Both can be unchecked at any time; unchecking falls back to v1.9.x's Stage-1-only behaviour. Settings UI: new "Research depth" card.

## New LlmStage tags

- `brand_research_verify` (Claude Opus, Stage 2 for brand)
- `brand_research_strategic` (Claude Opus, Stage 3 for brand)
- `product_research_verify` (Claude Opus, Stage 2 for product)
- `product_research_strategic` (Claude Opus, Stage 3 for product)

## Feedback wiring

`dossier_feedback.target_kind` already supports `'brand'` and `'product'` (added in v1.9.2 for forward-compat). v1.10.0 wires them up:

- `brands:research(id, { feedback? })` IPC extended
- `products:research(id, { feedback? })` IPC extended
- `FeedbackModal` (renamed from `SignalFeedbackModal`) parameterised for all 4 kinds
- BrandsProducts gains a `Re-research with feedback` button next to each existing research button

Feedback is injected into ALL three stages' prompts when present, so corrections shape research + verification + strategic intel coherently.

## UI changes (option a — inline)

`BrandResearchPanel` and product-card dossier rendering both gain:

1. **Confidence pills** next to each dossier field (green=high, amber=medium, red=low). Only shown when `confidence_levels` is populated; field still renders without pill if Stage 2 didn't run.
2. **"What we don't know" subsection** at the bottom of the dossier (rendered from `unknowns`). Hidden when null/empty.
3. **"Strategic Intelligence" collapsible section** below "What we don't know":
   - ICP segments rendered as a grid of cards (name, description, decision-maker, cycle length, key signals)
   - Buying-cycle scenarios + competitive plays as markdown
   - Hidden when `strategic_intel` is null.
4. **`Re-research with feedback` button** next to the existing `Run/Re-research` button on brand panel + per-product card.

## What stays the same

- Scanner (manual + deep scan) reads brand/product columns same way as before

- v1.10.0 fallback: if either advanced toggle is off, the entire pre-v1.10 flow runs
