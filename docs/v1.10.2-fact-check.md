# v1.10.2 — Stage 4 fact-check (fetch + verify cited sources)

**Status:** Spec'd 2026-05-27. Shipping in this session.

Final piece of the original v1.10 vision. Builds on v1.10.0's three-stage pipeline (Perplexity Stage 1 + Opus Stage 2 verify + Opus Stage 3 strategic).

## Why this exists

Stages 2+3 give Opus a verified-against-knowledge dossier, but Opus has no way to *fact-check* Stage 1's claims against the original web sources Perplexity cited. The user's stated quality bar was "zero hallucination as we can defensibly achieve" — that requires fetching the cited URLs and verifying claims against actual source text.

Stage 4 closes that loop.

## Architecture

```
researchBrand(id) / researchProduct(id)
  ├── Stage 1 — Perplexity sonar-deep-research        (unchanged)
  ├── Stage 2 — Claude Opus verify + sharpen          (v1.10.0)
  ├── Stage 3 — Claude Opus strategic intel           (v1.10.0)
  └── Stage 4 — Claude Opus fact-check               (NEW)
        INPUT: verified dossier + up to N fetched source texts
        STEPS:
          1. Pull up to factCheckMaxSources citations from Stage 1
          2. Fetch each via existing fetchUrl() (HTML strip, JS gracefully fails)
          3. Cap each source text at 8000 chars to bound token cost
          4. Send Opus: { dossier, sources[] } → ask for per-section verdict
        OUTPUT: fact_check_report JSON with:
          • per_section_verdicts: { section_name → { verdict, supporting_sources[], issues[] } }
          • flagged_claims: [{ claim, source_url, status, reason }]
          • overall_confidence: 'high' | 'medium' | 'low'
          • sources_fetched: how many of the N attempted actually succeeded
```

If `brandResearchFactCheck` / `productResearchFactCheck` setting is off, Stage 4 skipped — Stage 3 dossier remains as the final output. Anthropic key missing → skipped. If Stage 2 or 3 failed, Stage 4 is also skipped (it operates on verified dossier).

Stage 4 failure is non-fatal: Stage 1-3 output is preserved, fact-check section just doesn't appear.

## URL fetching reliability

Many sites block bots, require JS, or paywall content. Stage 4 must gracefully degrade:

- Use existing `fetchUrl()` from `knowledge.ts` (already strips HTML, handles redirects)
- Per-source timeout: 15 seconds (don't let one slow site hold up the whole pass)
- Parallel fetch with `Promise.allSettled` so one failure doesn't poison the batch
- Each source result: `{ url, text: string | null, error: string | null }`
- Sources that returned no text are skipped at Opus call time; their related claims are marked `inconclusive: source unreachable`

If <2 sources succeeded out of N attempted, log a warning and skip the Opus call entirely (not enough material to fact-check against). Status chip shows `Stage 4 skipped: only K/N sources fetched`.

## Token economics

Per call:
- Sources: up to 10 × 8K chars ≈ 80K input tokens
- Dossier + system prompt: ~5K input tokens
- Output: ~3K tokens of verification data
- Per call: ~\$1.30–1.80 Opus

For 10 brands + 30 products refreshed monthly with all 4 stages on: ~\$60-80/month. Negligible at user portfolio scale.

## Schema additions (idempotent)

On both `brands` and `products`:
- `fact_check_report TEXT NULL` — JSON output from Stage 4
- `last_fact_check_at TEXT NULL` — timestamp of last successful Stage 4

`research_status_detail` JSON gains `stage4` field tracking the same `pending | completed | skipped: <reason> | failed: <reason>` pattern.

## Settings

- `brandResearchFactCheck: boolean` (default `true`) — enables Stage 4 on brand research
- `productResearchFactCheck: boolean` (default `true`) — enables Stage 4 on product research
- `factCheckMaxSources: number` (default `10`, range 1-15) — cap on URLs fetched per call

UI: extend the "Research depth" card with two new toggles and the max-sources number input.

## New LlmStage tags

- `brand_research_factcheck` (Claude Opus, Stage 4 for brand)
- `product_research_factcheck` (Claude Opus, Stage 4 for product)

## UI changes

`BrandResearchPanel` and product dossier render gain:

1. **"Fact-check report" collapsible section** below "Strategic Intelligence". Shows:
   - Overall verdict chip (high/medium/low confidence) with source-coverage stat (e.g. "8/10 sources verified")
   - Per-section verdict cards (Positioning, Target ICP, Competitive Summary, Research Summary) with verdict + supporting sources list
   - Flagged claims table with status badges + reason + source link
   - Hidden when `fact_check_report` is null.
2. **ResearchStatusChip extended** to include Stage 4:
   - `Stage 1 ✓ · Stage 2 ✓ · Stage 3 ✓ · Stage 4 ✓` (all green)
   - `... · Stage 4 ⚠ K/N sources` (partial — some sources unreachable)
   - `... · Stage 4 ✗` (failed)
3. **"Opus verified + fact-checked" tag** on dossier summary line when both `last_advanced_research_at` and `last_fact_check_at` are set.
